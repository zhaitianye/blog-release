基本用法

凡是右边数据是有迭代器接口的数据，就都可以解构，常见的有迭代器接口的数据就是数组。 如果右边数据不是具有迭代器接口的数据，那就不能解构，会报错。 解构的顺序是从左到右，上面例子就是先解构第一个元素，赋值给第一个变量，然后向右进行。

复杂用法

解构赋值本质是模式匹配，只要等号两边的模式相同，左边的变量就会被赋予对应的值，如果不匹配，则只有匹配的几个变量会被赋值，其他变量默认为undefined。

基本用法

属性名匹配成功，就赋值，不成功就不赋值。不赋值默认也是undefined。

复杂用法

跟数组解构差不多，但是对象的解构赋值没有剩余符号的这一说。

引用不匹配的属性值

比如我的变量名跟对象的属性名恰好不一致，但确实希望变量得到该属性值，怎么办？就像这样，写成a:c，c是你写的变量名，a:c表示：虽然我变量名不叫a而是叫c，但是我希望c得到a属性的值。

深度赋值

看出端倪了么？其实， let { a, b } = { a: "aaa", b: "bbb" } 就可以理解为： let { a: a, b: b } = { a: "aaa", b: "bbb" } 然后对应赋值就可以了，冒号后面才是变量名，冒号前面就是“线索”，按照线索找值就行了。

对象解构赋值，不像数组解构赋值那样排排队，然后挨个赋值，而是每次都根据“线索”找值，能找到就找到，找不到就undefined。
