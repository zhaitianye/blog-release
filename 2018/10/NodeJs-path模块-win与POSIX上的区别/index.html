NodeJs-path模块-win与POSIX上的区别

2018-10-15
NodeJs
字数统计: 298   |   阅读时长≈ 1 分钟

path 模块的默认操作因 Node.js 应用程序运行所在的操作系统而异。 具体来说，当在 Windows 操作系统上运行时， path 模块将假定正在使用 Windows 风格的路径。 因此，使用 path.basename() 可能会在 POSIX 和 Windows 上产生不同的结果： 在 POSIX 上:

在 Windows 上:

path.win32.basename(‘C:\temp\myfile.html’);
// 返回: ‘myfile.html’

path.posix.basename(‘/tmp/myfile.html’);
// 返回: ‘myfile.html’

```

在 Windows 上，Node.js 遵循每个驱动器工作目录的概念。 当使用没有反斜杠的驱动器路径时，可以观察到此行为。 例如，
path.resolve('c:\\') 可能会返回与 path.resolve('c:') 不同的结果。
